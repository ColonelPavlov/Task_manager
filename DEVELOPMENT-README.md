# Как заставить этот ужас работать?

Это техническое чтиво для тех, кто решил развернуть этот проект у себя. Здесь написано, как развернуть оба сервиса у себя на компе, настроить окружение и ничего не сломать. 

---

## 1. Если вы работаете в VS Code

В корне проекта лежит файл **`Task_manager.code-workspace`**. Это мультикорневой воркспейс, который я настроил для удобства и кайфа. Он красиво разделит в боковой панели папки Core и Supporting и упростит управление двумя изолированными виртуальными окружениями, необходимыми для обеспечения бесперебойной работы сервисов Core и Supporting.

* **Как открыть:** В VS Code нажмите `File` -> `Open Workspace from File...` и выберите этот файл.
* **Это не обязательно!** Если вы работаете в PyCharm, WebStorm(?), Блокноте(??) или просто любите страдать и постоянно вручную прописывать пути в терминале, а потом судорожно проверять, завёлся ли `.venv`, то делайте как удобно. Воркспейс сделан чисто для вашего комфорта. (Ну и моего)

---

## 2. Настройка виртуальных окружений (Python) (Пуйтон)

У нас крутятся два независимых сервиса на разных фреймворках (FastAPI и Flask). Для каждого сервиса нужно создать **своё собственное** окружение и поставить свои библиотеки.

> 💡 **Примечание:** Окружение можно поставить абсолютно любым удобным для вас способом (вручную через терминал, средствами PyCharm и т.д.). Главное - соблюдать строгое правило ниже.

> ⚠️ **СТРОГОЕ ПРАВИЛО:** Создавать виртуальное окружение нужно внутри папки конкретного сервиса, и папка обязательно должна называться `.venv`. Если назвать её иначе, то `.gitignore` её пропустит.

### Ставим .venv в Core (FastAPI)
1. Зайдите терминалом в папку `Core(FastAPI)` (`cd Core`).
2. Создайте окружение:
   ```bash
   python -m venv .venv
   ```
3. Активируйте его и накатите ~~50г~~ библиотеки (они все уже лежат в `requirements.txt` этого сервиса):
   ```bash
   # Для Windows:
   .venv\Scripts\activate
   # Для macOS/Linux:
   source .venv/bin/activate

   pip install -r requirements.txt
   ```

### Ставим .venv в Supporting (Flask)
1. Перейдите в папку другого сервиса (`cd ../Supporting` или `cd Supporting`, если вы уже в корне проекта).
2. Создайте окружение и здесь:
   ```bash
   python -m venv .venv
   ```
3. Активируйте и поставьте все библиотеки (все они тоже лежат в `requirements.txt` уже этого сервиса):
   ```bash
   # Для Windows:
   .venv\Scripts\activate
   # Для macOS/Linux:
   source .venv/bin/activate

   pip install -r requirements.txt
   ```

## 3. Если вы доставляете новые библиотеки

Если в процессе разработки фичи вам понадобился какой-то новый пакет (я хрен знает для чего), обязательно обновите локальный список зависимостей в нужной папке перед тем, как делать коммит:
```bash
pip freeze > requirements.txt
```

Если VS Code вдруг начнет подчеркивать код красным и ругаться на импорты — нажмите `Ctrl+Shift+P` (или `Cmd+Shift+P`), вбейте `Python: Select Interpreter` и выберите тот интерпретатор, который лежит внутри `.venv` нужного сервиса.

## 4. Запуск обоих сервисов
```bash
# FastAPI (из папки Core/)
uvicorn app.main:app --reload --port 8000

# Flask (из папки Supporting/)
python main.py
```
